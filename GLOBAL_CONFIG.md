# 全域代理人設定 (Global Agent Configuration)

## 觸發 A — 新專案 (首次開啟)

當專案資料夾首次在 Antigravity 中開啟時，代理人必須執行：

1.  **讀取 Skills**：讀取 `C:\Users\2025.DESKTOP-7SF21LR\Documents\Agent_Skills\skills`。**若未找到任何 Skills，必須詢問使用者處理方式。**
2.  **驗證/建立 Junction**：檢查根目錄是否已有 `._agents/skills/Global_Skills` 的 Directory Junction。若無，建立指向全域技能目錄的 Junction。
3.  **確認與摘要**：
    *   提供讀取規則摘要 (例如：Dev_Workflow, Quality_Guard)。
    *   明示 Junction 路徑以確認執行成果。
4.  **確認任務**：詢問目前的任務或目標。

## 觸發 B — 現有專案 (後續對話)

在已配置過的專案開啟新對話時，代理人必須：

1.  **讀取 Skills**：重新讀取 `._agents/skills/Global_Skills` (若資、路徑遺失則改讀全域路徑)。**若未找到任何 Skills，必須詢問使用者處理方式。**
2.  **確認載入規則**：簡述啟用的規則 (例如：`[Dev_Workflow 啟用中] [Quality_Guard 啟用中]`)。
3.  **確認任務**：詢問目前任務或目標。不重複建立 Junction。

## 效益優先溝通原則

*   **效益最優先**：結果與價值導向。
*   **效率次之**：追求極簡、結論先行、無廢話。
*   **誠實告知**：不確定或推測時必須註明。禁止已知不確定時使用肯定語氣，避免浪費時間。
*   **拒絕中英並陳**：禁止不必要的中英雙語並陳，提升閱讀效率。
*   **語言標準**：繁體中文、台灣用語、禁止中國用語。專有名詞可直接用英文。

## 全域設定存入協議

收到「存入全域設定」指令時，必須執行兩件事：

1.  確實存入內容至 `GLOBAL_CONFIG.md`。
2.  提供當次設定點位摘要。

---

此協議確保專案維持開發流程與程式碼品質的高標準。
